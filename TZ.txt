Общие сведения1.1 Наименование системыНастоящее техническое задание предназначено для разработки веб-приложения «Система автоматизации обработки транзакций ПОС-терминалов с интеграцией REST API Межбанковского Процессингового Центра».1.2 Наименование исполнителяИсполнитель: студент группы ПИ(б)-3-24 Кыргызского Государственного Технического Университета имени И. Разакова.1.3 Краткие сведения о заказчикеЗаказчик: Halyk Bank Kyrgyzstan — коммерческий банк, осуществляющий операции на территории Кыргызской Республики. Банк предоставляет торговым организациям ПОС-терминалы для приёма безналичных платежей в валютах KGS, KZT, EUR и USD. Ежедневно через терминалы проходит значительный объём транзакций, данные о которых поступают от Межбанковского Процессингового Центра (МПЦ).1.4 Основание для разработкиОснованием для разработки системы является учебный план и договор между исполнителем и заказчиком. Проект выполняется в рамках учебной дисциплины «Разработка программного обеспечения».1.5 Плановые сроки начала и окончания работыНачало разработки: 01.03.2026Окончание разработки: 30.05.20262. Спецификация проблемыВ ходе изучения предметной области были выявлены следующие проблемы:Ручная обработка транзакционных данных. Сотрудники бухгалтерии ежедневно вручную открывают Excel-файлы, поступающие от МПЦ, и вручную переносят данные в банковскую систему. Этот процесс занимает 2–3 часа рабочего времени ежедневно.Высокий риск ошибок. При ручном вводе данных бухгалтер может перепутать сумму, валюту или код устройства, что приводит к некорректным проводкам, которые сложно обнаружить и исправить.Отсутствие автоматической агрегации. Бухгалтер вручную суммирует транзакции по каждому устройству и каждой валюте (DeviceCode + Currency) вместо автоматического расчёта.Ручное формирование отчётов. Формирование сводных отчётов за день, неделю или месяц выполняется вручную и требует значительных временных затрат.Отсутствие контроля дублирования. Система не предупреждает о повторной обработке одних и тех же транзакций, что может привести к задвоению проводок.Задержка получения данных. При ручном процессе данные от МПЦ обрабатываются только на следующий день, что не позволяет отслеживать операции в режиме реального времени.3. Цели и назначение создания3.1 ЦелиАвтоматизация получения транзакционных данных от МПЦ через REST API без участия сотрудников.Полное исключение ручного ввода данных и связанных с ним ошибок.Автоматическое формирование бухгалтерских проводок сразу после получения транзакций.Сокращение времени обработки ежедневных транзакций с 2–3 часов до нескольких минут.Обеспечение бухгалтера инструментом для мгновенного формирования отчётов за любой период.3.2 НазначениеРазрабатываемая система предназначена для:Автоматического запроса транзакционных данных у REST API МПЦ по расписанию.Хранения всех транзакций и проводок в базе данных PostgreSQL.Автоматического создания бухгалтерских проводок по каждой транзакции.Предоставления веб-интерфейса бухгалтеру для просмотра транзакций, проводок и отчётов.Формирования сводных отчётов с пересчётом сумм в национальную валюту (KGS).4.
